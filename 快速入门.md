# 快速入门 — 从源码构建

> **简而言之**：完整重建需要 **Bun**（而非 Node.js）以使用其编译时内置特性（`feature()`、`MACRO`、`bun:bundle`）。使用 esbuild 进行尽力而为的构建可以达到约 95% 的完成度，但需要手动修复约 108 个 feature-gate 模块。

**语言**：[English](QUICKSTART.md) | **中文**

---

## 方案 A：运行预构建 CLI（推荐）

npm 包中已包含编译好的 `cli.js`：

```bash
cd /path/to/parent/            # package.json 和 cli.js 所在目录
node cli.js --version           # → 2.1.88 (Claude Code)
node cli.js -p "你好，Claude"   # 非交互模式

# 或者全局安装：
npm install -g .
claude --version
```

**需要认证**：设置 `ANTHROPIC_API_KEY` 环境变量，或运行 `node cli.js login`。

## 方案 B：从源码构建（尽力而为）

### 前置要求

```bash
node --version   # >= 18
npm --version    # >= 9
```

### 步骤

```bash
cd claude-code-2.1.88/

# 1. 安装构建依赖
npm install --save-dev esbuild

# 2. 运行构建脚本
node scripts/build.mjs

# 3. 构建成功后运行：
node dist/cli.js --version
```

### 构建脚本的执行流程

| 阶段 | 操作 |
|------|------|
| **1. 复制** | `src/` → `build-src/`（原始文件保持不变） |
| **2. 转换** | `feature('X')` → `false`（启用死代码消除） |
| **2b. 转换** | `MACRO.VERSION` → `'2.1.88'`（编译时版本注入） |
| **2c. 转换** | `import from 'bun:bundle'` → 存根导入 |
| **3. 入口** | 创建注入 MACRO 全局变量的包装器 |
| **4. 打包** | esbuild 通过迭代存根创建处理缺失模块 |

### 已知问题

源码使用了 **Bun 编译时内置特性**，无法完全用 esbuild 复现：

1. **`feature('FLAG')` from `bun:bundle`** — Bun 在编译时将其解析为 `true`/`false` 并消除死代码分支。我们的转换替换为 `false`，但 esbuild 仍然会解析这些分支内的 `require()` 调用。

2. **`MACRO.X`** — Bun 的 `--define` 在编译时替换这些值。我们使用字符串替换，在大多数情况下有效，但在复杂表达式中可能遗漏。

3. **108 个缺失模块** — 这些是 feature-gate 的内部模块（守护进程、桥接助手、上下文折叠等），不存在于已发布的源码中。正常情况下 Bun 会将其死代码消除，但 esbuild 无法消除，因为 `require()` 调用在语法上仍然存在。

4. **`bun:ffi`** — 用于原生代理支持，已打存根。

5. **来自生成文件的 TypeScript `import type`** — 部分生成的类型文件不在已发布的源码中。

### 修复剩余问题

```bash
# 1. 检查还有哪些缺失：
npx esbuild build-src/entry.ts --bundle --platform=node \
  --packages=external --external:'bun:*' \
  --log-level=error --log-limit=0 --outfile=/dev/null 2>&1 | \
  grep "Could not resolve" | sort -u

# 2. 为每个缺失模块在 build-src/src/ 中创建存根：
#    JS/TS：创建导出空函数的文件
#    文本文件：创建空文件

# 3. 重新运行：
node scripts/build.mjs
```

## 方案 C：使用 Bun 构建（完整重建——需要内部访问权限）

```bash
# 安装 Bun
curl -fsSL https://bun.sh/install | bash

# 真实构建使用 Bun 的打包器配合编译时 feature flags：
# bun build src/entrypoints/cli.tsx \
#   --define:feature='(flag) => flag === "SOME_FLAG"' \
#   --define:MACRO.VERSION='"2.1.88"' \
#   --target=bun \
#   --outfile=dist/cli.js

# 但是，内部构建配置未包含在已发布的包中。
# 你需要访问 Anthropic 的内部仓库。
```

## 项目结构

```
claude-code-2.1.88/
├── src/                  # 原始 TypeScript 源码（1,884 个文件，512K 行代码）
├── stubs/                # Bun 编译时内置特性的构建存根
│   ├── bun-bundle.ts     #   feature() 存根 → 始终返回 false
│   ├── macros.ts         #   MACRO 版本常量
│   └── global.d.ts       #   全局类型声明
├── scripts/
│   └── build.mjs         # 构建脚本（基于 esbuild）
├── node_modules/         # 192 个 npm 依赖
├── vendor/               # 原生模块源码存根
├── build-src/            # 由构建脚本创建（转换后的副本）
└── dist/                 # 构建输出（由构建脚本创建）
```
