> 在深入阅读核心文档之前，本文帮助你建立必要的背景知识，理解仓库结构，并准备好学习环境。

## 一、什么是本仓库？

本仓库包含从 npm 包 `@anthropic-ai/claude-code` **v2.1.88** 中提取的反编译 TypeScript 源码，以及基于这份源码的深度分析文档。

**为什么这份源码有价值？**

Claude Code 是 Anthropic 开发的 AI 编程助手，它的源码揭示了：
1. **真实的系统提示词（System Prompt）**：Anthropic 如何指令 Claude 完成编程任务
2. **AI 行为控制机制**：如何设计多层次的 AI 行为约束
3. **提示词工程实战**：真实商业产品中的提示词设计技巧
4. **AI 产品架构**：大型 AI 产品的技术架构和设计决策

## 三、关键概念速览

在开始阅读前，先了解以下术语，这些术语在文档中频繁出现：

### 3.1 系统提示词（System Prompt）
AI 对话开始前设置的隐藏指令，用于定义 AI 的角色、行为规则和约束条件。例如：

```
你是一个专业的代码编辑助手。
你应该尽量简洁地回答。
不要在回复中包含不必要的解释。
```

### 3.2 Feature Flag（功能开关）
通过配置而非代码控制功能是否开启的机制。Claude Code 中所有 Feature Flag 都以 `tengu_` 开头，例如 `tengu_onyx_plover`（控制 Auto Dream 功能）。

### 3.3 遥测（Telemetry）
软件自动收集使用数据并发送给开发商的机制。Claude Code 会收集你的使用行为数据。

### 3.4 模型代号（Model Codename）
Anthropic 使用动物名作为内部模型代号：
- **Capybara（水豚）** → Sonnet 系列（当前版本 v8）
- **Tengu（天狗）** → 产品/遥测前缀
- **Fennec（耳廓狐）** → Opus 4.6 的前身
- **Numbat（袋食蚁兽）** → 下一代模型（未发布）

### 3.5 GrowthBook
一个 A/B 测试平台，Anthropic 用它对不同用户群体开启不同的功能，无需用户知情或同意。

### 3.6 KAIROS
Anthropic 内部代号，指即将推出的"自主代理模式"——AI 可以在后台自主工作、提交代码、发送通知，无需人类持续监督。

## 四、仓库结构详解

### 4.1 文档目录（`docs/`）

这是本仓库最重要的部分，包含五份深度分析报告：

```
docs/
├── en/   ← 英文原版（逻辑更完整，适合有英文基础的读者）
└── zh/   ← 中文翻译版（适合中文读者，内容完全一致）
```

### 4.2 源码目录（`src/`）

从 Claude Code npm 包中提取的 TypeScript 源码，共约 1,884 个文件，512K 行代码。关键子目录：

| 目录 | 内容 |
|------|------|
| `src/constants/` | **系统提示词**等常量定义 |
| `src/services/analytics/` | 遥测和分析服务 |
| `src/utils/` | 工具函数，包含 undercover.ts |
| `src/services/remoteManagedSettings/` | 远程设置管理 |
| `src/buddy/` | 虚拟宠物系统 |
| `src/tasks/` | 后台任务，包含 DreamTask |

### 4.3 其他重要文件

| 文件 | 说明 |
|------|------|
| `README.md` / `README_CN.md` | 仓库总概览（英文/中文） |
| `QUICKSTART.md` / `快速入门.md` | 如何构建和运行 Claude Code（英文/中文） |
| `scripts/excluded-strings.txt` | 被禁止出现在构建产物中的字符串（模型代号防泄漏） |
| `stubs/` | 构建时使用的占位模块 |

## 五、如何阅读源码（可选）

如果你想直接阅读最核心的提示词源码，推荐从这个文件开始：

```bash
# 在仓库根目录下执行：
# 搜索系统提示词中的关键内容
grep -n "UNDERCOVER MODE" src/utils/undercover.ts
grep -n "MODEL LAUNCH" src/constants/prompts.ts | head -20
```

或者直接查看完整的系统提示词常量文件（该文件较大，建议使用代码编辑器打开）：

```
src/constants/prompts.ts   ← 包含所有系统提示词文本
```

## 六、建立学习心态

### 你在学什么？
你不是在学习如何复制 Claude Code，而是在学习：
- **提示词工程的工业级实践**：真实商业产品中的提示词是如何设计的
- **AI 系统设计原则**：如何构建可靠、可控的 AI 产品
- **迁移到自己的工作流**：如何把这些原则应用到你使用的 Gemini 等工具上

### 建议的阅读方式
1. **先读中文文档**，建立整体理解
2. **标注关键提示词片段**，这些是你最宝贵的学习材料
3. **思考"为什么"**：为什么 Anthropic 要这样设计？这样设计解决了什么问题？
4. **联系自己的使用场景**：这个设计原则能用在我和 Gemini 的交互中吗？
