# 01 — 入门准备

> 在深入阅读核心文档之前，本文帮助你建立必要的背景知识，理解仓库结构，并准备好学习环境。

---

## 一、什么是本仓库？

本仓库包含从 npm 包 `@anthropic-ai/claude-code` **v2.1.88** 中提取的反编译 TypeScript 源码，以及基于这份源码的深度分析文档。

**为什么这份源码有价值？**

Claude Code 是 Anthropic 开发的 AI 编程助手，它的源码揭示了：
1. **真实的系统提示词（System Prompt）**：Anthropic 如何指令 Claude 完成编程任务
2. **AI 行为控制机制**：如何设计多层次的 AI 行为约束
3. **提示词工程实战**：真实商业产品中的提示词设计技巧
4. **AI 产品架构**：大型 AI 产品的技术架构和设计决策

- 编程经验（纯阅读文档不需要写代码）
- 深入的机器学习知识

## 三、关键概念速览

在开始阅读前，先了解以下术语，这些术语在文档中频繁出现：

### 3.1 系统提示词（System Prompt）
AI 对话开始前设置的隐藏指令，用于定义 AI 的角色、行为规则和约束条件。例如：

```
你是一个专业的代码编辑助手。
你应该尽量简洁地回答。
不要在回复中包含不必要的解释。
```

### 3.2 Feature Flag（功能开关）
通过配置而非代码控制功能是否开启的机制。Claude Code 中所有 Feature Flag 都以 `tengu_` 开头，例如 `tengu_onyx_plover`（控制 Auto Dream 功能）。

### 3.3 遥测（Telemetry）
软件自动收集使用数据并发送给开发商的机制。Claude Code 会收集你的使用行为数据。

### 3.4 模型代号（Model Codename）
Anthropic 使用动物名作为内部模型代号：
- **Capybara（水豚）** → Sonnet 系列（当前版本 v8）
- **Tengu（天狗）** → 产品/遥测前缀
- **Fennec（耳廓狐）** → Opus 4.6 的前身
- **Numbat（袋食蚁兽）** → 下一代模型（未发布）

### 3.5 GrowthBook
一个 A/B 测试平台，Anthropic 用它对不同用户群体开启不同的功能，无需用户知情或同意。

### 3.6 KAIROS
Anthropic 内部代号，指即将推出的"自主代理模式"——AI 可以在后台自主工作、提交代码、发送通知，无需人类持续监督。

### 4.3 源码目录（`src/`）— 核心代码

从 Claude Code npm 包中提取的 TypeScript 源码，约 **1,884 个文件，512K 行代码**。这是仓库中体积最大的部分。

#### 根级别关键文件

| 文件 | 大小 | 说明 |
|------|------|------|
| `src/main.tsx` | ~4,700 行 | **应用主入口**，包含整个 Claude Code 的 UI 渲染逻辑（使用 React/Ink 在终端渲染） |
| `src/QueryEngine.ts` | ~1,300 行 | **查询引擎**，负责将用户输入发送给 Claude API，处理流式响应 |
| `src/Tool.ts` | ~800 行 | **工具基类**，所有 AI 工具（文件编辑、命令执行等）的抽象基类 |
| `src/commands.ts` | ~750 行 | 所有 `/` 斜杠命令的注册和路由（如 `/help`、`/clear` 等） |
| `src/query.ts` | ~1,400 行 | 查询处理逻辑，含流式输出、工具调用循环 |
| `src/context.ts` | ~200 行 | 会话上下文管理 |
| `src/history.ts` | ~350 行 | 对话历史管理与持久化 |
| `src/cost-tracker.ts` | ~400 行 | API 调用费用追踪 |

### 4.4 构建脚本目录（`scripts/`）

| 文件 | 说明 |
|------|------|
| `build.mjs` | **主构建脚本**，将 TypeScript 源码打包为单一 `dist/cli.js`（使用 esbuild）|
| `prepare-src.mjs` | 构建前预处理：将 `src/` 复制到 `build-src/`，替换 Bun 特有语法 |
| `transform.mjs` | 代码转换：`feature('X')` → `false`，`MACRO.X` → 字面量值 |
| `stub-modules.mjs` | 自动为缺失的 feature-gate 模块生成空白存根文件 |

**构建流程**：`prepare-src.mjs` → `transform.mjs` → `stub-modules.mjs` → `build.mjs`（esbuild 打包）

---

### 4.5 存根目录（`stubs/`）

| 文件 | 说明 |
|------|------|
| `bun-bundle.ts` | 替代 `bun:bundle` 的存根：`feature()` 函数永远返回 `false`（禁用所有 Feature Flag） |
| `macros.ts` | 替代 Bun 编译时 `MACRO` 常量的运行时版本（版本号、构建时间等） |
| `macros.d.ts` | `MACRO` 全局变量的 TypeScript 类型声明 |
| `global.d.ts` | 全局类型声明文件 |

**为什么需要存根？** Bun 构建工具的编译时特性（`feature()`、`MACRO`）在 Node.js/esbuild 环境中不存在，需要用运行时替代品来让代码能够编译。

### 4.7 原生工具目录（`vendor/`）

包含 Claude Code 使用的**原生（非 JavaScript）模块**的源代码存根，这些模块在运行时以本机代码执行：

| 目录 | 说明 |
|------|------|
| `vendor/audio-capture-src/` | 音频捕获模块（支持语音输入功能） |
| `vendor/image-processor-src/` | 图像处理模块（支持图片分析） |
| `vendor/modifiers-napi-src/` | 键盘修饰键检测（用于快捷键系统） |
| `vendor/url-handler-src/` | URL 处理模块（处理 `claude://` 自定义协议） |

**注意**：这些是 C/C++/Rust 编写的原生模块的源代码，不是 TypeScript。它们通过 Node.js NAPI 接口与 JavaScript 代码交互。

## 六、建立学习心态

### 你在学什么？
你不是在学习如何复制 Claude Code，而是在学习：
- **提示词工程的工业级实践**：真实商业产品中的提示词是如何设计的
- **AI 系统设计原则**：如何构建可靠、可控的 AI 产品
- **迁移到自己的工作流**：如何把这些原则应用到你使用的 Gemini 等工具上

### 建议的阅读方式
1. **先读中文文档**，建立整体理解
2. **标注关键提示词片段**，这些是你最宝贵的学习材料
3. **思考"为什么"**：为什么 Anthropic 要这样设计？这样设计解决了什么问题？
4. **联系自己的使用场景**：这个设计原则能用在我和 Gemini 的交互中吗？
