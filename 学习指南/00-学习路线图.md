## 二、五份核心文档概览

这五份文档是本仓库最有价值的内容，按以下顺序阅读效果最好：

### 第一篇：遥测与隐私分析
- **文件**：`docs/zh/01-遥测与隐私分析.md`
- **核心主题**：Claude Code 如何收集用户数据，数据管道架构
- **对提示词设计的启示**：理解 AI 系统如何追踪和记录使用行为，为隐私感知的提示词设计提供参考
- **关键技术点**：
  - 双层分析管道（Anthropic 第一方 + Datadog 第三方）
  - `OTEL_LOG_TOOL_DETAILS=1` 环境变量可记录完整工具输入
  - 没有用户侧的退出开关

### 第二篇：隐藏功能与模型代号
- **文件**：`docs/zh/02-隐藏功能与模型代号.md`
- **核心主题**：内部模型代号体系，Feature Flag 设计，内外部用户差异
- **对提示词设计的启示**：**这是最重要的一篇**，揭示了 Anthropic 如何为不同用户设计不同的系统提示词
- **关键技术点**：
  - 内部员工获得更详细的输出（`"倾向于更多解释"`）
  - 外部用户被要求尽量简洁
  - Feature Flag 用随机词对（如 `tengu_onyx_plover`）掩盖真实用途

### 第三篇：卧底模式分析
- **文件**：`docs/zh/03-卧底模式分析.md`
- **核心主题**：Anthropic 员工在公开仓库中使用 AI 时的隐藏机制
- **对提示词设计的启示**：展示了一个完整的、实战使用的系统提示词（System Prompt）范例
- **关键技术点**：
  - 完整的 `UNDERCOVER MODE` 系统提示词文本（可直接学习其格式）
  - 归属系统（Attribution System）的设计
  - 模型代号屏蔽机制

### 第四篇：远程控制与紧急开关
- **文件**：`docs/zh/04-远程控制与紧急开关.md`
- **核心主题**：Anthropic 如何远程控制所有用户的 Claude Code 行为
- **对提示词设计的启示**：理解 AI 产品的控制层设计，学习如何构建有层次的 AI 行为约束系统
- **关键技术点**：
  - 每小时轮询远程设置
  - `接受否则退出` 对话框设计
  - 6+ 种紧急开关（Killswitch）

### 第五篇：未来路线图
- **文件**：`docs/zh/05-未来路线图.md`
- **核心主题**：Claude Code 即将推出的功能，包括 KAIROS 自主代理模式
- **对提示词设计的启示**：**KAIROS 的系统提示词是学习自主代理设计的绝佳案例**
- **关键技术点**：
  - KAIROS 自主代理系统提示词（`<tick>` 心跳机制）
  - 17 个未上线工具的设计思路
  - Buddy 虚拟宠物系统（娱乐设计参考）
